Question 1

What request type is common across all three (WMS, WFS, WCS) OGC web services that we have learned about?

The GetCapabilies request is common across all three web services.

Question 2

Answer the following questions about a WMS GetCapabilities request

  • What are the required parameters, and what do they represent?
  • What is returned in response to a WMS GetCapabilities request?

REQUEST=GetCapabilities tells the server the type of request. SERVICE=WMS tells the server this is a WMS request.

Question 3

Answer the following questions about a WMS GetMap request

  • What are the required parameters, and what do they represent?
  • What is returned in response to a WMS GetMap request?
  • What is the significance of transparency in WMS requests?

The required parameters are:

  • REQUEST=GetMap, Indentifies type of request
  • VERSION, current version
  • LAYERS, list of layers to return
  • SRS, Spatial Reference System
  • BBOX, bounding box coordinates of the map area
  • WIDTH, width of map area (pixels)
  • HEIGHT, height of map area (pixels)
  • STYLES, applies style to layer
  • FORMAT, type of file to return

If the GetMap request is formed correctly, an image of the map area will be returned. Transparency turns the background pixels invisible, so multiple layers can be viewed at the same time (i.e., roads on top of imagery). If transparency is not allowed then you will only see the top layer.
